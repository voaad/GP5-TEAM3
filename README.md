# GP5-TEAM3

## TMDB Movie Analytics: Предсказание рейтинга и анализ постеров

**Компания:** TMDB - крупнейшая онлайн-платформа каталога фильмов, сериалов и другого медиаконтента. Основной продукт TMDB: API, через которое сторонние разработчики (медиаплееры, мобильные приложения, стриминговые сервисы, сайты-агрегаторы) получают метаданные о фильмах и используют полученные данные в своих продуктах.

**Проблематика**: сейчас получаемые по эндпойнтам данные содержат фактическую информацию (средняя оценка пользователей, рейтинг, количество просмотров и тд.) Однако многие рекламные комании, использующие данные от TMDB также хотят оценить прогноз успешности телевезионных програм (и не только), но не имеют ресурсов для внедрения моделей скоринга и прогнозирования. Мы предлагаем добавить новые эндпойнты в API:

| Поле | Тип | Смысл | Для чего это бизнесу |
|------|-----|-------|-------------------|
| `predicted_rating` | float (0-10) | Предсказание средней оценки на основе метаданных (бюджет, жанр, язык, продолжительность, страна и т.д.) | Стриминговые сервисы (например, небольшие онлайн-кинотеатры) могут использовать это до выхода фильма, чтобы решить, какие фильмы покупать, какие продвигать.|
| `poster_genre_match` | float (0-1) или enum | Насколько постер соответствует основному жанру фильма (или конкретные флаги, например `poster_consistency_score`) | Приложения могут корректировать отображение, чтобы делать рекламу фильма привлекательнее и читаемее для клиента (к примеру, добавлять текстовую подсказку с реальным жанром) |

**Цель:** разработать два взаимосвязанных решения для платформы TMDB (The Movie Database) с использованием методов глубокого обучения:

1. **Табличная модель** - предсказание пользовательского рейтинга фильма на основе его метаданных (бюджет, сборы, жанр, язык, популярность и др.).
2. **Модель компьютерного зрения** - классификация жанра фильма по его постеру и выявление постеров, вводящих в заблуждение.

Обе модели работают на едином датасете и решают общую бизнес-задачу: повышение удовлетворённости пользователей и монетизации за счёт улучшения качества представления контента.

**Как TMDB может это монетизировать:**

- **Вариант 1:** Включить эти поля в Commercial Plan ($149/мес), таким образом повышая привлекательность платного тарифа.
- **Вариант 2:** Сделать отдельный адд-он **«ML Analytics Pack»** за дополнительные $50–100/мес.
- **Вариант 3:** Продавать доступ к этим полям через отдельный эндпоинт с поминутной тарификацией.

### Неструктурированные данные (постеры)

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `poster_path` | string | Относительный путь к постеру |
| `primary_genre` | string | Жанр |

**Данные для CV-модели:**
- Выбраны топ-7 жанров: Drama, Comedy, Action, Horror, Thriller, Crime, Adventure.
- Для каждого жанра взято по 1000 фильмов (балансировка, случайная выборка, `random_state=42`).
- Постеры скачаны по URL из API TMDB (предварительно сохранены в `data/posters/downloaded/`).


## Решаемые задачи и ML-решения

### Задача 1. Предсказание рейтинга (табличная модель)

**Тип задачи:** регрессия.

**Архитектуры:**
- Полносвязная сеть с 2 скрытыми слоями (64, 32 нейрона, ReLU)
- Увеличенная сеть (128, 64, 32, Dropout 0.3)
- Сеть с BatchNormalization и регуляризацией L2

**Метрики качества:** MAE, RMSE, R².

### Задача 2. Классификация жанра по постеру (CV)

**Тип задачи:** многоклассовая классификация (7 классов)

**Архитектуры (сравниваются не менее трёх):**
- Свёрточная сеть, построенная с нуля (3–4 свёрточных слоя + полносвязные)
- Трансферное обучение: **ResNet50** (ImageNet веса, дообучение)
- EfficientNet-B0 (лёгкая, высокая точность)

**Метрики качества:** accuracy, F1-score (macro), precision, recall, confusion matrix.
